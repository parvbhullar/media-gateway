    pub description: Option<String>,

    pub description: Option<String>,
    pub direction: Option<String>,
    pub priority: Option<i32>,
    pub is_active: Option<bool>,
    // NO `records` field — records-only-via-records-endpoints (D-04, D-27)
}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → POST /routing/tables | Untrusted JSON body crosses Bearer-auth into validators |
| operator → PUT /routing/tables/{name} | Same; `records` field MUST be rejected (D-04) |
| handler → SeaORM | Validated `serde_json::Value` records array → DB JSON column |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02-01 | Tampering | Records field on PUT-table-metadata | mitigate | `UpdateRoutingTableRequest` uses `#[serde(deny_unknown_fields)]` so any client sending `records` gets 400. Test `update_table_with_records_field_returns_400` proves it. |
| T-06-02-02 | Denial of Service | Initial `records` array on POST | mitigate | Hard cap of 1000 records per table enforced at validate time (D-discretion); rejected with 400. Prevents row-size blowup in JSON column. |
| T-06-02-03 | Denial of Service | Embedded JSON serialization | mitigate | Per-record JSON content not validated structurally in 06-02 (deferred to 06-03), BUT `validate_records_array` enforces `arr.len() <= 1000` and at-most-one-default. 06-03 layers per-record validation on top. |
| T-06-02-04 | Information Disclosure | List endpoint pagination | accept | Phase 6 ships unpaginated list (table count expected to be small — single-tenant operator scope). Pagination is a v2.1 follow-up if needed. |
| T-06-02-05 | Tampering | Race on POST duplicate name | mitigate | Rely on DB UNIQUE constraint on `name` column (Plan 06-01). Handler maps SQLSTATE 23505 to 409 Conflict. |
| T-06-02-06 | Spoofing | Bearer auth bypass | accept | Inherited from Phase 1 SHELL-01; new routes ride existing middleware unchanged. |
| T-06-02-07 | Repudiation | `created_at`/`updated_at` integrity | mitigate | Server-stamped via `Utc::now()` at insert/update; client cannot supply timestamps (not in request types). |
</threat_model>
