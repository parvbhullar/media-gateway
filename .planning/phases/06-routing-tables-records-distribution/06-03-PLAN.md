#[derive(Debug, Serialize, Deserialize, Clone)]
pub struct RoutingRecord {
    pub record_id: String,           // UUID v4 (server-generated on POST)
    pub position: i32,
    #[serde(rename = "match")]
    pub match_: RoutingMatch,
    pub target: RoutingTarget,
    #[serde(default)]
    pub is_default: bool,
    #[serde(default = "default_true")]
    pub is_active: bool,
}

#[derive(Debug, Serialize, Deserialize, Clone)]
#[serde(tag = "type", rename_all = "snake_case")]
pub enum RoutingMatch {
    Lpm { prefix: String },                         // D-08
    ExactMatch { value: String },                   // D-09
    Regex { pattern: String },                      // D-10
    Compare { op: CompareOp, value: CompareValue }, // D-11
    HttpQuery {
        url: String,
        #[serde(default)]
        timeout_ms: Option<u32>,
        #[serde(default)]
        headers: Option<HashMap<String,String>>,
    }, // D-12
}

#[derive(Debug, Serialize, Deserialize, Clone)]
#[serde(rename_all = "snake_case")]
pub enum CompareOp { Eq, Lt, Gt, In }

#[derive(Debug, Serialize, Deserialize, Clone)]
#[serde(untagged)]
pub enum CompareValue {
    Single(u32),
    Range([u32; 2]),
}

#[derive(Debug, Serialize, Deserialize, Clone)]
#[serde(tag = "kind", rename_all = "snake_case")]
pub enum RoutingTarget {
    TrunkGroup { name: String },              // D-24
    Gateway { name: String },
    NextTable { name: String },
    Reject { code: u16, reason: String },
}

#[serde(deny_unknown_fields)]

    pub is_active: bool,

}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → POST/PUT /records | Untrusted match + target JSON crosses Bearer auth into validator |
| handler → DB JSON column | Validated record array → `supersip_routing_tables.records` |
| handler → external HTTP at *match* time | NOT in this plan — write-time only validates URL shape; runtime HTTP call lives in 06-04 (its own threat block) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03-01 | Tampering | Operator-supplied regex pattern (D-10) | mitigate | `validate_routing_record` enforces `pattern.len() <= 4096` AND `regex::Regex::new(...)` compiles at write time. Caps regex DoS pattern complexity. Operator is authenticated, so this is hardening not a hostile-attacker boundary. |
| T-06-03-02 | Spoofing / SSRF | Operator-supplied HttpQuery URL (D-12) | mitigate | Reject non-http/https schemes; reject hosts in `{localhost, 127/8, 10/8, 172.16/12, 192.168/16, 169.254/16, ::1, fc00::/7, fe80::/10}`. Tests `post_http_query_url_localhost_returns_400`, `_private_ip_`, `_non_http_scheme_` enforce. Note: this is *write-time* defense; *runtime* defense (06-04) repeats the check before issuing the HTTP request to defend against DNS-rebind. |
| T-06-03-03 | Denial of Service | HttpQuery `timeout_ms` (D-16) | mitigate | Cap at 5000ms enforced at write time. Default 2000ms. |
| T-06-03-04 | Denial of Service | Embedded JSON column row size | mitigate | Hard cap 1000 records/table. Per-record fields are bounded (regex 4096, URLs reasonable). |
| T-06-03-05 | Tampering | Race on concurrent record edits | accept | Read-modify-write with no version check; last-write-wins. Acceptable for v2.0 — operator UI is single-user-per-table in practice. v2.1 candidate for `If-Match` ETag. |
| T-06-03-06 | Information Disclosure | Bearer auth | accept | Inherited from middleware. |
| T-06-03-07 | Repudiation | Default-record uniqueness invariant | mitigate | At most one `is_default: true` per table; enforced on POST + PUT. Test `post_two_defaults_returns_400` proves. |
| T-06-03-08 | Elevation of Privilege | Unknown match `type` / target `kind` slipping past serde | mitigate | Tagged-enum `#[serde(tag = "type")]` / `#[serde(tag = "kind")]` rejects unknown variants with deserialization error → 400. Tests `post_unknown_match_type_returns_400` and `post_unknown_target_kind_returns_400` prove. |
</threat_model>
