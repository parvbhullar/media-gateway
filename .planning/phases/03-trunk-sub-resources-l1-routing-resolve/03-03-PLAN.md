Validation helper (D-08):

```rust
fn validate_sip_uri(uri: &str) -> ApiResult<()> {
    let trimmed = uri.trim();
    if trimmed.is_empty() {
        return Err(ApiError::bad_request("uri must be non-empty"));
    }
    if trimmed.len() > 500 {
        return Err(ApiError::bad_request(
            "uri must be 1-500 chars (DB column limit)",
        ));
    }
    // D-08: parse via rsipstack URI parser. Same parser the proxy uses
    // for inbound INVITE Request-URIs, so a passing parse here means the
    // URI is dispatchable in the live call path.
    let parse_result: Result<rsipstack::sip::Uri, _> = trimmed.try_into();
    match parse_result {
        Ok(_) => Ok(()),
        Err(e) => Err(ApiError::bad_request(format!(
            "invalid SIP URI: {}",
            e
        ))),
    }
}
```

```rust
async fn list_uris(
    State(state): State<AppState>,
    Path(name): Path<String>,
) -> ApiResult<Json<Vec<TrunkOriginationUriView>>> {
    let db = state.db();
    let trunk_group_id = lookup_trunk_group_id(db, &name).await?;
    let rows = TouEntity::find()
        .filter(TouColumn::TrunkGroupId.eq(trunk_group_id))
        .order_by_asc(TouColumn::Position)
        .all(db).await
        .map_err(|e| ApiError::internal(e.to_string()))?;
    Ok(Json(rows.into_iter().map(TrunkOriginationUriView::from).collect()))
}

async fn add_uri(
    State(state): State<AppState>,
    Path(name): Path<String>,
    Json(req): Json<AddTrunkOriginationUriRequest>,
) -> ApiResult<(StatusCode, Json<TrunkOriginationUriView>)> {
    let db = state.db();
    validate_sip_uri(&req.uri)?;
    let trunk_group_id = lookup_trunk_group_id(db, &name).await?;

    // Pre-check duplicate (UNIQUE (trunk_group_id, uri) per D-06)
    let dup = TouEntity::find()
        .filter(TouColumn::TrunkGroupId.eq(trunk_group_id))
        .filter(TouColumn::Uri.eq(req.uri.clone()))
        .one(db).await
        .map_err(|e| ApiError::internal(e.to_string()))?;
    if dup.is_some() {
        return Err(ApiError::conflict(format!(
            "uri '{}' already exists on trunk '{}'", req.uri, name
        )));
    }

    // D-06: auto-assign position = MAX(position) + 1, or 0 for first.
    let next_position: i32 = TouEntity::find()
        .filter(TouColumn::TrunkGroupId.eq(trunk_group_id))
        .order_by_desc(TouColumn::Position)
        .one(db).await
        .map_err(|e| ApiError::internal(e.to_string()))?
        .map(|r| r.position + 1)
        .unwrap_or(0);

    let now = Utc::now();
    let am = trunk_origination_uris::ActiveModel {
        trunk_group_id: Set(trunk_group_id),
        uri: Set(req.uri.clone()),
        position: Set(next_position),
        created_at: Set(now),
        ..Default::default()
    };
    let inserted = am.insert(db).await
        .map_err(|e| ApiError::internal(e.to_string()))?;

    Ok((StatusCode::CREATED, Json(TrunkOriginationUriView::from(inserted))))
}

async fn delete_uri(
    State(state): State<AppState>,
    Path((name, uri)): Path<(String, String)>,
) -> ApiResult<StatusCode> {
    let db = state.db();
    // axum URL-decodes path segments by default, so {uri} arrives decoded.
    let trunk_group_id = lookup_trunk_group_id(db, &name).await?;
    let row = TouEntity::find()
        .filter(TouColumn::TrunkGroupId.eq(trunk_group_id))
        .filter(TouColumn::Uri.eq(uri.clone()))
        .one(db).await
        .map_err(|e| ApiError::internal(e.to_string()))?
        .ok_or_else(|| {
            ApiError::not_found(format!(
                "uri '{}' not found on trunk '{}'", uri, name
            ))
        })?;
    TouEntity::delete_by_id(row.id).exec(db).await
        .map_err(|e| ApiError::internal(e.to_string()))?;
    Ok(StatusCode::NO_CONTENT)
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> /api/v1/trunks/{name}/origination_uris | Bearer-authenticated; URI in path segment is URL-decoded by axum; URI in JSON body is parsed via rsipstack SIP URI parser |
| handler -> rsipstack URI parser | Trust boundary: parser rejects malformed input but might accept exotic-but-valid SIP URI shapes — accept since the same parser handles live INVITEs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-URI-01 | Tampering | URI in path segment for DELETE | mitigate | URL-decoded by axum; URI is treated as opaque string — no shell interpretation. The DB column is VARCHAR(500), preventing pathological-length truncation attacks. The rsipstack parser is what eventually consumes the URI in production; URIs failing validate_sip_uri on POST cannot enter the table. |
| T-03-URI-02 | Information Disclosure | Wire response includes the full URI | accept | URIs are operator-configured destinations; no PII leakage. Same trust level as the trunk_group itself. |
| T-03-URI-03 | Denial of Service | Auto-assigned position via MAX+1 | accept | Operator-driven CRUD; per-trunk max position is bounded by reasonable carrier deployments (~tens of URIs). DB-level UNIQUE constraint prevents duplicate-flooding. Race: two concurrent POSTs could both compute the same `next_position`, but the UNIQUE constraint on `(trunk_group_id, uri)` prevents the duplicate insert; one would succeed and the other would 500 on the DB error (acceptable for operator workflows). |
| T-03-URI-04 | Tampering | XSS via URI string in error response | mitigate | Error responses are JSON; client-side XSS only possible if a console UI renders the response as HTML without escaping. The console UI uses templated rendering with HTML escaping by default — verified via Phase 1 patterns. Not a server-side concern. |
</threat_model>
