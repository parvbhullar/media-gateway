    Replace 07-01 stub with full impl:
    ```rust
    use dashmap::DashMap;
    use tokio_util::sync::CancellationToken;

    /// In-memory registry of cancellation tokens for in-flight webhook delivery
    /// retries. Per D-31 (cancel-on-DELETE) and D-34 (PUT replaces prior token).
    #[derive(Default)]
    pub struct WebhookCancelRegistry {
        inner: DashMap<String, CancellationToken>,
    }

    impl WebhookCancelRegistry {
        pub fn new() -> Self { Self::default() }

        /// Insert a fresh CancellationToken for `id`. If an entry already exists
        /// (PUT-replace per D-34), the prior token is cancelled before insert.
        pub fn insert(&self, id: &str) -> CancellationToken {
            let token = CancellationToken::new();
            if let Some((_, prior)) = self.inner.remove(id) {
                prior.cancel();
            }
            self.inner.insert(id.to_string(), token.clone());
            token
        }

        /// Cancel the stored token for `id` and remove the entry. No-op if
        /// missing. Per D-31 — DELETE handler triggers this.
        pub fn cancel(&self, id: &str) {
            if let Some((_, token)) = self.inner.remove(id) {
                token.cancel();
            }
        }

        /// Remove without cancelling. Used after natural delivery success.
        pub fn remove(&self, id: &str) {
            self.inner.remove(id);
        }

        pub fn contains_key(&self, id: &str) -> bool {
            self.inner.contains_key(id)
        }
    }
    ```

    Add `#[cfg(test)] mod tests` covering the 7 behavior cases above. Use `tokio::spawn` + `JoinSet` for the concurrent case; assert `registry.inner.len() == 100` after `JoinSet::join_all`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| signer input | Operator-controlled secret + arbitrary JSON body; no untrusted data crosses into HMAC computation beyond what the operator already controls |
| cancel_registry | Process-local memory only; no IPC, no persistence (D-33 explicit) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-03-01 | Tampering | HMAC algorithm choice | mitigate | SHA-256 (industry standard, Stripe-pattern). HMAC-SHA1 explicitly NOT supported. Custom algorithms deferred to v2.1 (CONTEXT.md §Deferred). |
| T-07-03-02 | Repudiation | Signature replay | mitigate (partial) | Stripe-style `{timestamp}.{body}` pattern lets RECEIVERS reject signatures > 5min old. Server-side sender does NOT replay-protect itself; documented in operator deploy guide as a receiver responsibility. |
| T-07-03-03 | Information Disclosure | secret memory residency | accept | Plaintext String per D-35; passed through `as_bytes()` to HMAC then dropped at scope-end. No long-term memory pinning. Whole-process memory dump is operator-deploy concern. |
| T-07-03-04 | Denial of Service | Unbounded body size in sign() | mitigate | sign() is O(n) over body length; 07-04 caps body indirectly via reqwest payload (CallRecord JSON is bounded by SeaORM row size). No streaming hash needed at v2.0 scale. |
| T-07-03-05 | Tampering | Cancel registry race | mitigate | DashMap provides atomic remove-or-insert via `remove()` returning Option. PUT-replace cancels old token BEFORE inserting new — invariant: no two valid tokens exist for same id. |
| T-07-03-06 | Denial of Service | Registry leak (cancel without insert) | mitigate | Each insert pairs with one of: cancel (DELETE), remove (delivery success), or process-restart (D-33 — in-memory only). No accumulation across restarts. |
| T-07-03-07 | Elevation of Privilege | Token forgery | mitigate | CancellationToken is process-local; can't be forged externally. Cancel only reachable via authenticated DELETE /webhooks/{id} (07-05 wires this). |
</threat_model>
