D-13 grammar — three valid forms:
- `allow all`, `deny all`
- `allow 1.2.3.4`, `deny 10.0.0.5` (single IPv4 or IPv6)
- `allow 1.2.3.0/24`, `deny 2001:db8::/32` (CIDR IPv4 or IPv6)

Suggested validator (uses std `IpAddr`/`Ipv4Addr` parsing — no new dep):
```rust
pub fn validate_acl_rule(rule: &str) -> Result<(), String> {
    let trimmed = rule.trim();
    let mut parts = trimmed.splitn(2, ' ');
    let action = parts.next().unwrap_or("");
    let target = parts.next().unwrap_or("").trim();
    let action_lc = action.to_ascii_lowercase();
    if action_lc != "allow" && action_lc != "deny" {
        return Err(format!("rule must start with 'allow' or 'deny': {}", rule));
    }
    if target.is_empty() {
        return Err(format!("rule missing target after action: {}", rule));
    }
    if target.eq_ignore_ascii_case("all") { return Ok(()); }
    if target.contains('/') {
        // CIDR — validate via ipnet or by hand-splitting
        let mut ts = target.splitn(2, '/');
        let ip = ts.next().unwrap();
        let prefix: u8 = ts.next().and_then(|p| p.parse().ok())
            .ok_or_else(|| format!("invalid CIDR prefix: {}", target))?;
        let parsed: std::net::IpAddr = ip.parse()
            .map_err(|_| format!("invalid CIDR address: {}", target))?;
        let max = if parsed.is_ipv4() { 32 } else { 128 };
        if prefix > max {
            return Err(format!("CIDR prefix {} exceeds max for family", prefix));
        }
        return Ok(());
    }
    let _ip: std::net::IpAddr = target.parse()
        .map_err(|_| format!("invalid IP literal: {}", target))?;
    Ok(())
}
```

(If `ipnet` is already a workspace dep, prefer `ipnet::IpNet::from_str`. Check `Cargo.toml`
during implementation; if not present, use the std-only form above to avoid a new dep —
matches the project convention from CONTEXT.md D-07.)

Default policy = allow (D-14) — operator MUST add `deny all` last to flip to default-deny; document this in the handler doc-comment. |
| T-05-03-03 | Information Disclosure | rule listing | accept | ACL rules are operator config, not secrets. Auth-gated. |
| T-05-03-04 | Denial of Service | unbounded POSTs | mitigate | UNIQUE (trunk_group_id, rule) caps duplicates. Volume cap is operator policy; no programmatic limit in v2.0. |
| T-05-03-05 | Tampering | DELETE bypassing 404 | mitigate | DELETE strictly looks up `(trunk_group_id, rule)` before delete and 404s if missing — no silent no-op that could hide drift. |
| T-05-03-06 | Tampering | URL-decoded rule mismatch | mitigate | axum's Path extractor URL-decodes once; the test suite includes a round-trip assertion (POST then DELETE the same rule) that proves decoding parity. |
