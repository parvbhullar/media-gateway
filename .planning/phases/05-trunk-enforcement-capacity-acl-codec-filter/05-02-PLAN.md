## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /api/v1 | Authenticated operator submits capacity values; flow into DB |
| DB → handler | trusted; SeaORM owns parameterization |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Tampering | PutTrunkCapacityRequest | mitigate | `#[serde(deny_unknown_fields)]` rejects unexpected fields. `validate_capacity` rejects 0 (D-05) and serde rejects negative integers because the field type is `u32`. Resulting DB column is `Option<i32>`; cast u32 → i32 is safe for our domain (max_calls ≤ 2³¹-1 is the same constraint operators face). |
| T-05-02-02 | Spoofing | parent-missing 404 | mitigate | `lookup_trunk_group_id` is the single resolver; returns 404 before any read/write of capacity table. Mirrors trunk_credentials.rs precedent. |
| T-05-02-03 | Denial of Service | unbounded PUT calls | accept | PUT is upsert — at most one row per trunk_group (UNIQUE FK from Plan 05-01); no growth vector. Auth layer caps abuse rate. |
| T-05-02-04 | Information Disclosure | GET response | accept | `current_active` and `current_cps_rate` are operator-relevant capacity telemetry, not PII. Exposed only to authenticated bearer-token holders. |
| T-05-02-05 | Repudiation | created_at/updated_at on capacity row | mitigate | Timestamps written by handler on insert + update so PUT history is at least last-write traceable. |
| T-05-02-06 | Elevation of Privilege | bypassing auth | mitigate | Router is mounted under the `protected` builder in mod.rs by Plan 05-01 (same place as Phase 3 sub-routers); inherits Bearer auth middleware. Test 1 verifies 401 without auth. |
</threat_model>
