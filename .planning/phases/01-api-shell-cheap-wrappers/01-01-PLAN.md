Pure-fn signature rule (CONTEXT.md "Adapter Pattern"):
```rust
pub(crate) async fn <action>(
    db: &DatabaseConnection,
    <typed_input>,
) -> Result<<typed_output>, DbErr>
```

both the HTML handler (which still owns `State<Arc<ConsoleState>>`) and the
new JSON handler (which owns `State<AppState>`) call the same fn.
