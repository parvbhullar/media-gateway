<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SIP hot path -> SecurityModule | Untrusted IP processed before auth and acl modules |
| REST PATCH /security/firewall -> DB + cache | Operator input replaces entire ruleset atomically |
| REST DELETE /security/blocks/{ip} -> DB + cache | Unblock operation must invalidate in-memory cache |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02-01 | Elevation of Privilege | DELETE /security/blocks/{ip} | mitigate | Endpoint is behind Bearer-token auth middleware (applied in mod.rs Layer); only authenticated operators can unblock |
| T-10-02-02 | Tampering | PATCH /security/firewall cache gap | mitigate | Cache updated ONLY after DB transaction commits (RISK-06 mitigation); in-process cache serves old rules during gap |
| T-10-02-03 | Denial of Service | SecurityModule DashMap entry creation | mitigate | Flush task evicts stale WindowState entries (window expired > 2x duration) to bound memory under spoofed-IP flood |
| T-10-02-04 | Spoofing | SecurityModule IP extraction | mitigate | Via-header extraction copied verbatim from acl.rs — consistent method prevents IP mismatch across modules |
| T-10-02-05 | Repudiation | Flood/auth-failure events | accept | In-memory counters survive up to flush_interval (30s) data loss on crash; acceptable for operational stats in Phase 10 |
</threat_model>
