Replace the Plan 02-01 create_trunk stub with a full handler. Add the supporting types above the router fn.

**1. DTO definitions** — CreateTrunkRequest with deny_unknown_fields, required fields name + members, optional display_name / direction / distribution_mode / credentials / acl / nofailover_sip_codes / is_active (default true via fn default_true). CreateTrunkMember substruct with gateway_name required + optional weight + priority.

**2. Validation helper** fn validate_trunk_group_name(name: &str) -> ApiResult<()>:
- Trim, assert non-empty, assert len <= 64, assert every char is alphanumeric or underscore or hyphen.
- Error message: "trunk group name must be 1-64 chars, alphanumeric plus _ or -".
- Use manual char iteration — do not pull in the regex crate unless it is already a direct dependency (check first via `grep -E "^regex" Cargo.toml`).

**3. fn validate_distribution_mode(mode: TrunkGroupDistributionMode) -> ApiResult<()>**:
- If mode is TrunkGroupDistributionMode::Parallel and feature parallel-trunk-dial is NOT enabled, return ApiError::bad_request("parallel distribution requires the parallel-trunk-dial feature").
- Use `#[cfg(not(feature = "parallel-trunk-dial"))]` on the reject branch so the code compiles both with and without the feature.

**4. Gateway existence helper** async fn validate_gateway_refs(db, members) -> ApiResult<()>:
- Short-circuit if members is empty → bad_request "trunk group must have at least one member".
- Collect gateway_name strings, run a single IN query against sip_trunk::Entity with Column::Name.is_in(names).
- Subtract found from requested → missing list.
- If missing is non-empty → bad_request with message "unknown gateway(s): name1, name2".

**5. Name collision helper** async fn assert_no_gateway_name_collision(db, name) -> ApiResult<()>:
- Query sip_trunk::Entity by Name.eq(name). If Some → bad_request "trunk group name 'N' collides with existing gateway".

**6. create_trunk handler body** — execute validations in this exact order (matches CONTEXT.md §Validation sequence):
  1. validate_trunk_group_name(req.name)
  2. derive direction = req.direction.unwrap_or_default()
  3. derive mode = req.distribution_mode.unwrap_or_default(); validate_distribution_mode(mode)
  4. validate_gateway_refs(db, req.members)
  5. assert_no_gateway_name_collision(db, req.name)
  6. Duplicate trunk_group check: TrunkGroupEntity::find().filter(TrunkGroupColumn::Name.eq(req.name)). If Some → conflict "trunk group 'N' already exists".
  7. Begin tx via db.begin(). Build trunk_group::ActiveModel (name, display_name, direction, distribution_mode, credentials, acl, nofailover_sip_codes, is_active, metadata=None, created_at=now, updated_at=now). insert(&tx) to get the row back.
  8. For idx, member in req.members.iter().enumerate(): build trunk_group_member::ActiveModel (trunk_group_id=inserted.id, gateway_name, weight.unwrap_or(100), priority.unwrap_or(0), position=idx as i32), insert(&tx).
  9. tx.commit().
  10. Reload members via TrunkMemberEntity::find().filter(TrunkGroupId.eq(inserted.id)).all(db).
  11. Return (StatusCode::CREATED, Json(view_from(inserted, members))).

**Rollback semantics:** SeaORM DatabaseTransaction auto-rolls-back on Drop when commit is not called. Using `?` inside the tx block therefore rolls back automatically on the first error. Do not add manual tx.rollback() inside map_err closures — it would require moving the tx value out of scope.

**3. Engagement check helper** async fn engagement_check_trunk_group(db, name) -> ApiResult<()>:

**4. delete_trunk handler**:
- Load existing by name → 404 on miss.
